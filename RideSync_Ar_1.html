Single endpoint for both roles. Body always has username, fullname, email, password, role. If role is "driver", body also includes vehicle: { make, model, color, year, plate, vehicleType, capacity }.
role: "rider"
→ createUser()

→ Return tokens

→ createUser()

→ Return tokens

Login — POST /api/users/login (one endpoint, all roles)
Finds user by email, verifies password with isPasswordCorrect(), generates both tokens. JWT payload carries role — this is what the frontend uses to route.

Every API request → middleware chain
Your existing authUser and role middlewares handle the first two checks. The new requireVerifiedCaptain middleware adds document + expiry validation on ride routes.

allowCaptainOnly → req.user.role === "driver" ✓ or 403
requireVerifiedCaptain → captain.captainVerified === "verified"
                                → vehicle.vehicleVerified === "verified"
                                → insurance.expiryDate > Date.now()
                                → puc.expiryDate > Date.now()

                                → captain.isActive === true
                                → attach req.captain → next()

Captain & Vehicle Verification States
Both Captain and Vehicle have independent verification fields. A captain can only set isActive = true when both are "verified" AND no documents are expired.
pending
Just registered. Docs uploaded. isActive: false. App shows "Account under review".
under_review
Admin opened docs in dashboard. Captain gets push notification. Still blocked.
verified
All docs pass. Admin sets verified. System auto-sets isActive: true. Captain can drive.
rejected
Doc problem. Admin fills verificationNote. Captain sees reason, re-uploads.
expired
Cron job flips status. isActive: false. Captain notified to renew expired document.

Middleware Chain on Ride Routes
The new requireVerifiedCaptain middleware sits third in the chain — after role check, before the handler. It populates the vehicle in one query and validates all conditions, returning a specific error message per failure.
// verify.middleware.js — new file
export const requireVerifiedCaptain = async (req, res, next) => {
  const captain = await Captain.findOne({ user: req.user._id }).populate("vehicle")
  if (!captain) return res.status(404).json({ message: "Captain not found" })
  if (captain.captainVerified !== "verified")
    return res.status(403).json({ message: "Account not verified yet" })
  if (!captain.isActive)
    return res.status(403).json({ message: "Account is inactive" })
  const v = captain.vehicle
  if (!v || v.vehicleVerified !== "verified")
    return res.status(403).json({ message: "Vehicle not verified" })
  if (new Date(v.insurance.expiryDate) < Date.now())
    return res.status(403).json({ message: "Insurance expired — please renew" })
  if (new Date(v.puc.expiryDate) < Date.now())
    return res.status(403).json({ message: "PUC certificate expired" })
  if (new Date(captain.license.expiryDate) < Date.now())
    return res.status(403).json({ message: "Driving licence expired" })
  req.captain = captain // attach for controllers to use
  next()
}

Daily Expiry Cron Job
// cron/checkExpiry.js — runs every day at midnight using node-cron
import cron from 'node-cron'
import { Vehicle } from '../models/vehicle.model.js'
import { Captain } from '../models/captain.model.js'

cron.schedule('0 0 * * *', async () => {
  const now = new Date()

  // 1. Find vehicles with expired insurance
  const expiredInsurance = await Vehicle.find({
    "insurance.expiryDate": { $lt: now },
    "insurance.status": { $ne: "expired" }
  })
  for (const v of expiredInsurance) {
    v.insurance.status = "expired"
    await v.save()
    await Captain.findOneAndUpdate(
      { vehicle: v._id },
      { isActive: false }
    )
    // TODO: send push notification to captain
  }

  // 2. Find vehicles with expired PUC (same pattern)
  // 3. Find captains with expired driving licence (same pattern)
})
